Detailed notes on bridging the gap between Dev and Ops: Culture, Linux, Git, and CI/CD Pipelines.

DevOps

1. What is DevOps?

DevOps is a set of practices and a culture that combines software development (Dev) and IT operations (Ops) to automate and integrate processes.

Key Aspects

  • Culture & Collaboration: Shared responsibility; breaking down silos.
  • Automation: Reducing manual errors in building/testing.
  • Integration: Single continuous process from planning to ops.
  • Continuous Delivery: Frequent, high-quality releases.

Benefits

  • Faster Delivery: Quick turnaround.
  • Increased Reliability: Stable software via automated testing.
  • Improved Efficiency: Streamlined processes.
  • Customer Value: Continuous value delivery.

3. Linux File System Explained

"In Linux, everything is a file." Understanding the directory tree is key to system administration.

/ (Root): The top-level directory. Everything starts here.
/bin (Binary): Essential binary executables (boot/repair).
/home User home directories (personal space).
/etc (Etcetera): Configuration files and system-wide settings.
/var (Variable): Logs, databases, mail. Data that changes often.
/usr (User): User-related programs, libraries, documentation.
/tmp (Temporary): Files that auto-clear.
/opt (Optional): Optional software packages.
/mnt (Mount): For external devices (USB, network shares).
/dev (Devices): Special files representing hardware.
Tip: Need to reset a password? Check resources like shellscript.sh or local admin guides for MySQL/MariaDB resets.

5. CI/CD: Seamless Deployment

Continuous Integration & Continuous Delivery.

Continuous Integration (CI)

Regularly merging code changes into a central repo. Example: You change a Python function; CI ensures it's brought back to the main codebase and tested automatically.

Continuous Delivery (CD)

Automated deployment. Once merged and tested, code is deployed to production without manual intervention.

Tools Comparison

Jenkins

Industry Standard

Open-source, highly extensible with plugins. "If you can dream it, you can build it."

Downside:

Requires self-hosting. You manage the server, build agents, security, and Java runtime. High overhead for small teams.

GitHub Actions

Cloud Native

Integrated directly into GitHub. Easier to set up for existing repos.

Upside:

Less maintenance. No need to manage a build server.
