Topic Overview

Introduction to Terraform

An open-source Infrastructure as Code (IaC) tool that lets you define both cloud and on-prem resources in human-readable configuration files.

Infrastructure as Code

1. What is Terraform?

Terraform enables you to build, change, and version infrastructure safely and efficiently. It creates and manages resources on cloud platforms (AWS, Azure, GCP) and other services through their APIs.

Providers

Terraform uses plugins called "Providers" to interact with remote systems. There are thousands of providers for everything from AWS and Kubernetes to GitHub and Datadog.

Declarative Configuration

You describe the end state you want (e.g., "I need 5 VMs"), and Terraform figures out the logic to reach that state (e.g., "Create 2 more VMs").

2. The Core Workflow

The standard lifecycle for managing infrastructure.

  • 1. Write
    Define resources across providers. For example, creating a VPC with security groups and load balancers.
  • 2. Plan
    Terraform creates an execution plan describing what it will create, update, or destroy to match your config.
  • 3. Apply
    On approval, Terraform executes the proposed operations in the correct order.

Terraform uses HashiCorp Configuration Language (HCL).

  region = "us-west-2"

  tags = {
    Name = "ExampleAppServerInstance"
  }
