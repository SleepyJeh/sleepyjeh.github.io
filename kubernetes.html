Topic Overview

Kubernetes (K8s)

Container orchestration platform that manages containerized applications across a cluster of machines for scalability and resilience.

Container Orchestration

1. Docker vs. Kubernetes

While often used together, they serve different purposes in the stack.

Docker

The Runtime

A platform to package applications with dependencies and run them on a single machine.

Kubernetes

The Orchestrator

A platform to manage containerized apps across a cluster of machines. It handles scaling, failover, and networking.

2. Cluster Architecture

A K8s cluster consists of a Control Plane (Master) and a set of Worker Nodes.

Control Plane Components (The Brain)
kube-apiserver
The front end. Exposes the Kubernetes API and handles all requests.
etcd
Consistent, highly-available key-value store for all cluster data.
kube-scheduler
Watches for new Pods with no assigned node and selects a node for them to run on.
kube-controller-manager
Runs controller processes (Node Controller, Job Controller, EndpointSlice Controller).
cloud-controller-manager
Links your cluster into your cloud provider's API (AWS, GCP, Azure).
Node Components (The Workers)
kubelet
An agent running on each node. It ensures that containers are running in a Pod.
kube-proxy
Maintains network rules on nodes. Allows network communication to your Pods.
Container Runtime
Software capable of running containers (e.g., containerd, CRI-O).
