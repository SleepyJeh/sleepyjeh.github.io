Topic Overview

Docker Fundamentals

A platform for developing, shipping, and running applications. Docker separates your applications from your infrastructure using containers.

Containerization

1. What is Docker?

Docker packages software into standardized units called containers that have everything the software needs to run including libraries, system tools, code, and runtime.

The Problem

"It works on my machine!"

Developers often struggle with different environments (OS versions, library dependencies) between dev, test, and production.

The Solution

Isolation & Portability

Docker containers run exactly the same way on any machine that has Docker installed, regardless of the underlying OS.

2. Core Architecture

Docker uses a Client-Server architecture.

Component Function
Docker Client The CLI tool (docker command) you use to issue commands. It talks to the daemon via REST API.
Docker Host (Daemon) The background service (dockerd) that manages images, containers, networks, and storage volumes.
Docker Registry Stores Docker images. Docker Hub is the default public registry where you can find images like Nginx, Python, etc.

3. Key Concepts

Image vs. Container

Think of an Image as a Class (OOP) or a Recipe. It is read-only.
Think of a Container as an Object or the Cake. It is a running instance.

Data & Network

  • Volumes: Persist data even if the container is deleted. Data is stored on the host file system.
  • Networks: Allow containers to talk to each other securely. (e.g., Web container talking to DB container).

4. The Dockerfile

A text document that contains all the commands to assemble an image.

FROM node:18-alpine     # Base image
WORKDIR /app            # Set working directory
COPY package.json .     # Copy dependency definitions
RUN npm install         # Install dependencies
COPY . .                # Copy app source code
EXPOSE 3000             # Document the port
CMD ["npm", "start"]    # Command to run on start

5. Cheat Sheet

docker pull [image] Download image from registry.
docker build -t [name] . Build an image from Dockerfile.
docker run -d -p 80:80 [image] Run container in background, mapping ports.
docker ps List running containers.
docker stop [id] Stop a running container.
docker rm [id] Remove a stopped container.
